# 多功能Base编解码器

## &#x20;项目简介

这是一个基于 **Python tkinter** 开发的 **多功能 Base 编解码工具**，提供直观的图形界面，支持将任意文件或文本转换为 **5 种不同的 Base 编码格式**，以及将编码数据还原为原始文件。

**支持的编解码算法：**
- **Base2 (二进制)**：由 0 和 1 组成的二进制字符串
- **Base8 (八进制)**：由 0-7 组成的八进制字符串
- **Base16 (十六进制)**：由 0-9, a-f 组成的十六进制字符串
- **Base32**：标准 RFC 4648 Base32 编码
- **Base64**：标准 RFC 4648 Base64 编码

## 功能特性

### 加密（编码）功能

- **多算法支持**：通过下拉框选择 5 种编码算法（Base2/Base8/Base16/Base32/Base64）
- **文件模式**：选择任意文件，将其内容转换为指定格式的编码字符串
- **文本模式**：直接输入文本，转换为对应的编码数据
- **流式处理**：采用分块读取和写入，支持大文件处理（已测试 100MB+ 文件）
- **实时预览**：显示前 100 位编码数据，方便确认转换结果
- **智能导出**：自动生成带时间戳或原文件名的输出文件
- **动态统计**：根据不同算法自动调整统计单位（位/八位组/字符）

### 解密（解码）功能

- **多算法解码**：支持 5 种算法的解码操作，自动识别并还原原始数据
- **文件输入**：选择包含编码数据的 `.txt` 文件进行解密
- **文本输入**：直接粘贴或输入编码字符串进行解码
- **自动清理**：自动过滤空格、换行符等无关字符
- **输入验证**：根据所选算法自动验证输入数据的有效性
- **多编码支持**：优先使用 UTF-8 解码，失败时自动降级到 Latin-1

### 配置管理功能

- **路径记忆**：自动记住上次加密/解密的保存目录，下次打开时自动定位
- **user.ini 配置文件**：程序首次运行时自动生成，存储用户偏好设置
- **ConfigManager 类**：统一的配置管理接口，支持读写操作

### 界面特性

- **标签页布局**：加密/解密功能分页展示，清晰直观
- **算法选择下拉框**：加密和解密页面均提供算法选择控件
- **实时统计**：显示总位数、已处理位数、剩余位数、处理速度（单位随算法变化）
- **进度条反馈**：大文件处理时显示实时进度百分比
- **状态提示**：成功/错误状态用颜色区分（绿色/红色）
- **剪贴板支持**：小文件可一键复制结果到剪贴板（≤8KB）

### 性能优化

- **多线程架构**：后台线程处理耗时操作，界面不卡顿
- **内存优化**：流式读写，避免一次性加载大文件到内存
- **智能更新**：小文件瞬间完成，大文件才启用进度条更新

## 使用说明

### 加密流程（文件/文本 → Base编码）

1. 打开程序，默认在「加密」标签页
2. **选择算法**：在处理区的下拉框中选择要使用的编码算法（Base2/Base8/Base16/Base32/Base64）
3. 选择输入方式：
   - **选择文件**：点击"选择文件"按钮，选取要编码的文件
   - **输入文本**：切换到"输入文本"模式，在文本框中输入内容
4. 点击 **"转换后导出为文档"** 按钮
5. 选择保存位置和文件名（会自动定位到上次使用的目录）
6. 等待处理完成，查看预览区和统计信息

### 解密流程（Base编码 → 原始数据）

1. 切换到「解密」标签页
2. **选择算法**：在处理区的下拉框中选择与加密时相同的算法
3. 选择输入方式：
   - **选择编码文件**：点击按钮选择之前生成的 `.txt` 文件
   - **手动输入**：在文本框中粘贴编码字符串
4. 点击 **"导出解密为文档"** 按钮
5. 选择输出路径和文件名（会自动定位到上次解密的目录）
6. 完成后即可获得原始文件

### 小技巧

- **快速复制**：对于小数据（≤8KB），可直接点击"复制到剪贴板"
- **大文件警告**：超过 100MB 的文件会弹出确认提示
- **文件名智能识别**：解密时自动去除 `.txt` 后缀作为默认输出名
- **实时验证**：输入内容变化时自动检测有效性（根据算法校验字符合法性）
- **路径记忆**：程序会自动记住你上次保存文件的位置，下次打开对话框时自动定位到该目录
- **单位自适应**：统计信息会根据选择的算法自动调整显示单位（Base2 显示"位"，Base8 显示"八位组"，其他显示"字符"）

## &#x20;注意事项

1. **非专业加密工具**：这只是编码转换工具，不具备真正的加密安全性（但比明文强一点点...大概吧？）
2. **文件体积膨胀**：不同算法的膨胀倍数如下：
   - **Base2 (二进制)**：约 **8 倍**（1 字节 → 8 字符）
   - **Base8 (八进制)**：约 **3 倍**（1 字节 → 3 字符）
   - **Base16 (十六进制)**：约 **2 倍**（1 字节 → 2 字符）
   - **Base32**：约 **1.6 倍**（每 5 字节 → 8 字符）
   - **Base64**：约 **1.33 倍**（每 3 字节 → 4 字符）
3. **内存限制**：虽然支持流式处理，但极端大的文件仍需注意系统资源
4. **编码问题**：特殊字符可能在不同系统间存在兼容性差异（已内置 UTF-8/Latin-1 双重降级机制）
5. **算法匹配**：**加密和解密必须使用相同算法**，否则无法正确还原数据！
6. **仅限学习娱乐**：请勿用于正式的安全场景！（虽然我觉得也没人会用这个去加密机密文件吧...）
